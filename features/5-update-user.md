# Update user

Provide a way for a user to update that user's data.

## Implementation

User updates are accomplished using the endpoint PUT /api/user.

The request header must contain the Authorization header, with a value of "Token {token}", where {token} is the jwt token.

If the authorization header is missing, return status code 401 and the response:
```json
{
  "errors": {
    "token": ["is missing"]
  }
}
```

The request payload looks like:
```json
{
  "user":{
    "email": "jake@jake.jake",
    "bio": "I like to skateboard",
    "image": "https://i.stack.imgur.com/xHWG8.jpg",
    "username": "Jake",
    "password": "new password"
  }
}
```

All the user fields in the above payload are optional, but at least one user field must be provided in the request.

Only the bio and image fields can be set to "" or to null.

If a non-nullable field is set to null or to "", return http status 422 and an error response informing the user that the field can't be blank. For example:
```json
{
  "errors": {
    "username": ["can't be blank"]
  }
}
```

If the username or email are already taken by another user, return http status 409 and an error about the duplicate data. For example:
```json
{
  "errors": {
    "email": ["has already been taken"]
  }
}
```

If the token does not match to any user, return status code 401 and response
```json
{
  "errors": {
    "credentials": ["invalid"]
  }
}
```

If the token matches to a user, apply the updates to that user record in the db.

A successful response returns status code 200 and the updated user details contained in the json generated by webserver.UserResponse.
